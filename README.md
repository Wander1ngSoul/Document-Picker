## 📖 Описание

**Protocol Builder** — это инструмент для автоматизации создания протоколов. Программа извлекает данные, обрабатывает их и генерирует единый протокол с точным форматированием согласно установленным стандартам.

### Для чего используется:
- ✅ Автоматический сбор данных из таблиц
- ✅ Конвертация устаревших `.doc` файлов в `.docx`
- ✅ Фильтрация и обработка информации о призывниках
- ✅ Генерация итогового документа с 8 колонками
- ✅ Применение точного форматирования (шрифты, размеры, выравнивание)

---

## ⚡ Возможности

- **Поддержка форматов**: Работа с `.doc` и `.docx` файлами
- **Автоматическая конвертация**: Прозрачная конвертация `.doc` → `.docx` через MS Word
- **Умный парсинг**: Автоматическое распознавание таблиц и извлечение данных
- **Фильтрация**: Поиск строк по ключевым словам (ФИО, год рождения, фамилии)
- **Точное форматирование**: 
  - Шрифт Times New Roman (14pt для данных, 10pt для номеров)
  - Фиксированная ширина колонок
  - Альбомная ориентация A4
  - Поля 1.5 см
- **Гибкая настройка**: Легко изменить параметры оформления и ключевые слова

## 💻 Системные требования

| Компонент | Требование |
|-----------|------------|
| **Операционная система** | Windows 10/11 (обязательно) |
| **Python** | Версия 3.8 или выше |
| **Microsoft Word** | Установленный MS Word (2010 или новее) |
| **Оперативная память** | Минимум 2 GB RAM |
| **Свободное место** | 100 MB на диске |

### ⚠️ Важные замечания

1. **Только Windows**: Скрипт использует `win32com.client` для конвертации `.doc` файлов
2. **Microsoft Word обязателен**: Без установленного Word конвертация не работает
3. **Разрядность**: Разрядность Python должна совпадать с разрядностью MS Office

## 📥 Установка

### Шаг 1: Установка Python

1. Скачайте Python с официального сайта: https://www.python.org/downloads/
2. **Важно**: При установке отметьте ✅ **"Add Python to PATH"**
3. Нажмите "Install Now"

### Шаг 2: Проверка установки

Откройте командную строку (`Win + R` → `cmd` → `Enter`):

```cmd
python --version
```

Должно отобразиться: `Python 3.x.x`

### Шаг 3: Создание виртуального окружения (рекомендуется)

```cmd

venv\Scripts\activate
```

После активации перед путём появится `(venv)`.

### Шаг 4: Установка зависимостей

```cmd
pip install python-docx pywin32
```

Дождитесь завершения установки.

## 🚀 Быстрый старт

### 1. Запуск программы

```cmd
python run.py
```

### 2. Выбор файлов

- Откроется окно выбора файлов
- Выделите нужные `.doc` или `.docx` файлы (можно несколько через `Ctrl`)
- Нажмите **"Открыть"**

### 3. Обработка

В консоли появится прогресс:

```
Выбрано файлов: 2

Обработка файла: C:\Users\...\file1.doc
Конвертация .doc в .docx...

Обработка файла: C:\Users\...\file2.docx

Всего собрано строк данных: 2

Создание протокола с 2 строками данных
```

### 4. Сохранение

- Откроется окно **"Сохранить как"**
- Укажите имя файла (по умолчанию: `Протокол №8 от 19.04.2025.docx`)
- Выберите папку
- Нажмите **"Сохранить"**

### 5. Завершение

- Автоматически откроется папка с готовым файлом
- В консоли нажмите `Enter` для выхода

## 📘 Подробная инструкция

### Подготовка исходных файлов

**Требования к документам:**

1. **Формат**: `.doc` или `.docx`
2. **Структура**: Документ должен содержать таблицу
3. **Количество колонок**: Минимум 8 колонок
4. **Содержание**: Во второй колонке должны быть:
   - Фамилия, имя, отчество
   - Год рождения (формат: `2003г.р.` или `г.р.`)
   - Или ключевые фамилии из списка

**Пример правильной структуры:**

| № п/п | Фамилия И.О., 2003г.р. ... | Диагноз ... | Жалобы ... | ... |
|-------|---------------------------|-------------|------------|-----|
| 1 | Иванов Иван Иванович, 2003г.р. ... | ... | ... | ... |

### Как работает программа

#### Этап 1: Выбор и конвертация

1. Открывается диалог выбора файлов
2. Для каждого `.doc` файла:
   - Запускается MS Word в фоновом режиме
   - Файл конвертируется во временный `.docx`
   - Оригинал не изменяется
3. `.docx` файлы открываются напрямую

#### Этап 2: Извлечение данных

Программа анализирует каждую таблицу:

1. **Фильтрация таблиц**: Ищутся таблицы с 8+ колонками
2. **Пропуск заголовков**: Игнорируются строки с:
   - `"№ п/п"` в первой колонке
   - `"Фамилия"` во второй колонке
3. **Пропуск числовых строк**: Игнорируются строки только с цифрами
4. **Поиск данных**: Строка считается данными, если во второй колонке есть:
   - `"г.р."` или `"рождения"`
   - Или одна из фамилий: `Казаков`, `Насиров`, `Кирилл`, `Зиганшин`, `Клипов`, `Сафиуллин`, `Тухтаев`

#### Этап 3: Создание протокола

1. **Настройка страницы**:
   - Формат: A4 (29.7 × 21.0 см)
   - Ориентация: Альбомная
   - Поля: 1.5 см со всех сторон

2. **Создание таблицы**:
   - 8 колонок с фиксированной шириной
   - Заголовки (шрифт 14pt, по центру)
   - Нумерация колонок 1-8 (шрифт 10pt, по центру)
   - Данные (шрифт 14pt)

3. **Применение стилей**:
   - Шрифт: Times New Roman
   - Размер: 14pt (данные), 10pt (номера)
   - Межстрочный интервал: одинарный
   - Отступы ячеек: 0 см
   - Вертикальное выравнивание: по центру

### Описание файла `run.py`

| Компонент | Назначение |
|-----------|------------|
| `DocumentMerger` | Главный класс программы |
| `select_files()` | Выбор файлов через диалог |
| `extract_data()` | Извлечение данных из таблиц |
| `convert_doc_to_docx()` | Конвертация `.doc` → `.docx` |
| `create_protocol()` | Создание итогового документа |
| `create_table()` | Формирование таблицы с данными |
| `_format_cell()` | Применение стилей к ячейкам |
| `set_column_widths()` | Установка ширины колонок |
| `cleanup_temp_files()` | Удаление временных файлов |
